1. O Grande Salto: Do Imperativo ao Declarativo

Para entender o Jetpack Compose, primeiro precisamos desaprender como fazíamos interfaces no Android Clássico (XML). No modelo antigo, trabalhávamos de forma imperativa. Imagine que você é um pintor e precisa descrever cada passo: "pegue o pincel, molhe no azul, faça um círculo na coordenada X,Y". Se o círculo precisasse mudar de cor, você teria que voltar lá, apagar e pintar de novo.

🔴 Era XML (Imperativa)

Você manipula manualmente a árvore de views. Se o dado muda, você precisa buscar o componente (findViewById) e setar o novo valor (setText). É propenso a erros de sincronia.

🟢 Era Compose (Declarativa)

Você descreve o que quer ver na tela baseado no estado atual. "Se o usuário está logado, mostre a foto dele". O Compose redesenha a UI automaticamente sempre que o estado muda.

Cenário Real: Carrinho de Compras

No XML, para atualizar o preço total, você teria que ouvir cada clique, somar e atualizar o TextView. No Compose, você apenas diz: Text("Total: $soma"). Quando a lista de itens muda, o Compose percebe e atualiza o texto sozinho. Isso é reatividade.

2. A Ciência da Recomposição

O conceito mais profundo do Compose é a Recomposição. Quando um dado usado por uma função @Composable muda, essa função é executada novamente com o novo dado.

No entanto, o Compose é inteligente: ele não redesenha a tela inteira (o que seria lento). Ele usa um algoritmo de diffing para identificar exatamente qual pedaço da interface depende daquele dado e atualiza apenas o necessário. Isso garante uma performance de 60 FPS (quadros por segundo) mesmo em telas complexas.

O Fluxo da UI Moderna

ESTADO Os dados (ex: Lista de Preços)
➔
COMPOSABLE A função que consome o dado
➔
UI O que o usuário vê na tela

3. Anatomia Técnica: Funções e Anotações

Toda interface no Compose começa com uma função Kotlin comum, mas com "superpoderes" conferidos por anotações específicas:

  • @Composable: Esta não é apenas uma etiqueta. Ela avisa ao compilador do Kotlin que esta função vai emitir nós de UI. Funções Composable só podem ser chamadas de dentro de outras funções Composable.
  • @Preview: É o fim do ciclo "compilar -> esperar -> ver no celular". Com ela, você renderiza sua UI diretamente no IDE. Você pode ter vários previews: um para tema escuro, outro para tela pequena, etc.
// Anatomia de um componente profissional
@Composable
fun BotaoCustomizado(texto: String) {
  Button(onClick = { /* ação */ }) {
    Text(text = texto)
  }
}

@Preview(name = "Visualização Padrão")
@Composable
fun PreviewBotao() {
  BotaoCustomizado("Clique Aqui")
}

4. Layouts: Organizando o Espaço

Diferente do XML, onde tínhamos ConstraintLayout para tudo, no Compose usamos três componentes fundamentais que resolvem 90% dos casos:

⫴

Column

Empilha elementos verticalmente (um abaixo do outro).

⫫

Row

Organiza elementos horizontalmente (um ao lado do outro).

❏

Box

Sobrepõe elementos (um em cima do outro, como camadas).

💡 Dica de Especialista: A Ordem dos Modifiers

O Modifier é o objeto que usamos para estilizar componentes. Um segredo que confunde iniciantes: a ordem importa!

Modifier.padding(8.dp).background(Color.Red)

Resultado: O preenchimento acontece PRIMEIRO, e o fundo vermelho aparece em volta do conteúdo original.

Modifier.background(Color.Red).padding(8.dp)

Resultado: O fundo fica vermelho PRIMEIRO, e o padding empurra o conteúdo para dentro desse fundo vermelho.

Laboratório: Construindo o Perfil do Motorista

Neste laboratório, vamos construir um card de identificação para o App de Logística da Fatec Garça. O objetivo é praticar o aninhamento de Column e Row.

1

Preparando o Terreno

Abra o Android Studio e crie um projeto Empty Compose Activity. Apague o código de exemplo do Greeting.

2

O Componente Composable

Digite o código abaixo. Note como usamos Modifier para dar forma ao card.

@Composable
fun DriverProfileCard(nome: String, veiculo: String) {
  // Card é um container com sombra e bordas arredondadas
  Card(
    modifier = Modifier.padding(16.dp).fillMaxWidth(),
    elevation = 4.dp
  ) {
    Row(modifier = Modifier.padding(16.dp)) {
      // Simulação de Foto
      Box(modifier = Modifier.size(60.dp).background(Color.LightGray))
      
      Spacer(modifier = Modifier.width(16.dp))
      
      Column {
        Text(text = nome, style = MaterialTheme.typography.h6)
        Text(text = "Veículo: $veiculo", color = Color.Gray)
      }
    }
  }
}
3

Visualizando sem Celular

Adicione a função de Preview abaixo e clique em Build & Refresh na aba lateral.

@Preview(showBackground = true)
@Composable
fun PreviewDriver() {
  DriverProfileCard("Ricardo Santos", "Scania R450")
}
