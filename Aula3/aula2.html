01. A Árvore de Composição

Diferente do Android antigo, onde você tinha objetos Java de botões e textos, no Compose **não existem objetos de View persistentes**. A tela é desenhada por uma sucessão de chamadas de funções.

O que é uma Função Composable?

É uma função marcada com @Composable. O compilador do Kotlin a transforma em um emissor de nós. Ela não retorna nada (é Unit), mas "emite" uma descrição do que deve aparecer.

@Composable
fun MyText() { Text("Olá") }

Slot API: A Engenharia de Encaixe

O Compose usa o conceito de **Slots**. Componentes como Button ou Card têm "buracos" (slots) onde você pode injetar qualquer outro Composable. Isso permite flexibilidade infinita sem herança complexa.

03. A Ciência do Modifier: Como a Tela se Transforma

O Modifier é o objeto mais importante do Compose. Ele é lido **sequencialmente**. Se você mudar a ordem, mudará o resultado visual.

1. fillMaxWidth() / size()

Define o tamanho que o componente "pede" ao pai. É a primeira restrição.

2. background()

Pinta a área definida até este momento no código.

3. padding()

Cria um espaço interno. Se houver um background antes, o padding fica *dentro* da cor. Se for depois, ele "empurra" o background.

4. clickable()

Define a área de interação. Se você colocar o clickable antes do padding, o usuário só consegue clicar no conteúdo, não na margem.

Visualização de Código vs Resultado

Modifier
  .size(100.dp)
  .background(Color.Red)
  .padding(10.dp)
  .background(Color.Blue)

Resultado: Um quadrado vermelho de 100dp com um quadrado azul de 80dp centralizado dentro dele.

04. Construção Real: App Gourmet Garça

Vamos construir o card de produto e o formulário, explicando **exatamente** o que cada linha faz na tela do dispositivo.

🚀 Parte A: O Card de Produto (Arquitetura de Slot)

@Composable
fun ProductCard(name: String, price: Double) {
  // 1. O Card cria a "elevação" (sombra) e arredondamento na tela.
  Card(
    modifier = Modifier.fillMaxWidth().padding(8.dp),
    elevation = 4.dp,
    shape = RoundedCornerShape(12.dp)
  ) {
    // 2. Row organiza a foto e os textos HORIZONTALMENTE.
    Row(
      modifier = Modifier.padding(16.dp),
      verticalAlignment = Alignment.CenterVertically
    ) {
      // 3. Box simula a imagem. clip() corta o fundo em círculo.
      Box(
        modifier = Modifier.size(50.dp)
          .clip(CircleShape)
          .background(Color.LightGray)
      )
      
      // 4. Spacer cria o "respiro" entre a foto e o texto.
      Spacer(modifier = Modifier.width(16.dp))
      
      // 5. Column organiza Nome e Preço VERTICALMENTE.
      Column(modifier = Modifier.weight(1f)) {
        Text(text = name, fontWeight = FontWeight.Bold)
        Text(text = "R$ $price", color = Color.Gray)
      }
    }
  }
}

📝 Parte B: O Formulário Reativo (Gestão de Estado)

Aqui explicamos por que a tela "sabe" que você digitou algo.

@Composable
fun RegistrationForm() {
  // 1. O ESTADO: "remember" guarda o valor na memória da tela.
  // Se não usar remember, o valor zera toda vez que a tela redesenhar.
  var nome by remember { mutableStateOf("") }

  Column(modifier = Modifier.padding(16.dp)) {
    // 2. O COMPONENTE: O campo de texto lê o valor da variável 'nome'.
    OutlinedTextField(
      value = nome,
      // 3. A REAÇÃO: 'it' é o que o usuário digitou agora.
      // Quando atualizamos 'nome', o Compose força a REDESENHAR esta função.
      onValueChange = { nome = it },
      label = { Text("Nome do Cliente") },
      modifier = Modifier.fillMaxWidth()
    )
  }
}
