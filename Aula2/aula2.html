1. Anatomia das Variáveis: val vs var

Diferente de linguagens onde a mutabilidade é o padrão, o Kotlin incentiva o design imutável para evitar efeitos colaterais em aplicativos concorrentes (muito comum no Android).

val (Value) - A Referência de Leitura Única

Em nível de compilação, o val gera apenas um getter. É similar ao final no Java ou const no JS. Use para constantes, configurações e dados que não devem ser alterados após o cálculo inicial.

// Imutável: Garantia de segurança em Threads
val pi: Double = 3.14159
val appName = "Logistics App" // Tipo inferido como String

var (Variable) - O Contêiner Mutável

Gera um getter e um setter. Essencial para contadores, estados de UI (como o texto digitado em um formulário) ou buffers de dados.

// Mutável: O valor pode ser reatribuído
var contador: Int = 0
contador += 1 // Incremento válido

2. Expressões: Tudo Retorna Algo

Conceito Formal: Em Kotlin, 'if' e 'when' não são apenas instruções (statements), são expressões.

If como Expressão

val status = if (nota >= 6) "Aprovado" else "DP"

Isso elimina a necessidade do operador ternário (a?b:c).

When: O Switch com Esteroides

val msg = when (nota) {
  10 -> "Perfeito!"
  in 7..9 -> "Muito bom"
  else -> "Estude mais"
}

3. Funções: Os Blocos de Construção

As funções em Kotlin são cidadãs de primeira classe. Elas podem ter valores padrão nos parâmetros, o que reduz drasticamente a sobrecarga de métodos comum no Java.

fun formatarMoeda(valor: Double, simbolo: String = "R$"): String {
  return "$simbolo $valor"
}

// Chamada 1: Usa padrão
println(formatarMoeda(150.0)) // Saída: R$ 150.0
// Chamada 2: Sobrescreve padrão
println(formatarMoeda(50.0, "US$")) // Saída: US$ 50.0

Laboratório: Roteiro Prático

Siga as fases abaixo no seu Scratch File do Android Studio.

Fase 1: Manipulação de Tipos e Strings

Objetivo: Entender a interpolação e declaração explícita.

val usuario: String = "Ferdinando"
var saldo: Double = 1250.50
println("Olá $usuario, seu saldo é $saldo")

Fase 2: Lógica Condicional de Negócio

Objetivo: Aplicar regras de negócio dinâmicas.

val idade = 17
val podeDirigir = if (idade >= 18) true else false

val categoriaHabilitacao = when {
  idade < 16 -> "Não permitido"
  idade in 16..17 -> "Apenas acompanhado"
  else -> "Habilitado"
}

Fase 3: Refatoração para Funções

Objetivo: Organizar o código para reuso.

fun avaliarCredito(renda: Double, score: Int): Boolean {
  return renda > 2000.0 && score > 600
}

val aprovado = avaliarCredito(2500.0, 700)
println("Resultado Crédito: $aprovado")

Exercícios Básicos

  1. 1. Declare uma variável imutável com o valor do Salário Mínimo e uma mutável com seu Salário Atual. Calcule quantos salários mínimos você ganha.
  2. 2. Crie uma String Template que exiba seu nome em maiúsculas (Dica: use ${nome.uppercase()}).
  3. 3. Escreva uma função que receba um Int e retorne se ele é positivo, negativo ou zero.
  4. 4. Use um for simples para imprimir todos os números pares de 2 a 100.
  5. 5. Crie uma função de uma única linha (Single-expression function) que converta Celsius para Fahrenheit.

Exercícios Intermediários

  1. 6. Sistema de Desconto: Crie uma função que receba o valor da compra. Se > 500, desconto de 20%. Se > 200, desconto de 10%. Use when.
  2. 7. Verificador de Triângulos: Função que recebe 3 lados e diz se é Equilátero, Isósceles ou Escaleno.
  3. 8. Simule um caixa eletrônico: Receba um valor de saque e verifique se há notas de 100, 50, 20 e 10 suficientes (use lógica de divisão e resto).
  4. 9. Crie uma função que receba uma lista de nomes e imprima apenas os que começam com a letra 'A'.
  5. 10. Implemente uma função que calcule o Fatorial de um número usando um loop while.
